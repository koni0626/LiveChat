# カメラ反応判定機能

## 目的

ユーザーが明示的にカメラをONにした場合だけ、キャラクター発話後にプレイヤーの表情を1枚だけ一時撮影し、楽しそう・迷っていそう・困っていそうなどの弱い反応フィードバックを得る。

画像そのものは保存しない。AI判定後に破棄し、セッション状態には判定結果だけを保存する。

## 方針

- デフォルトOFF
- ユーザーが明示的にONにする
- カメラON状態を画面に常時表示する
- いつでもOFFにできる
- キャラクター発話後だけ撮影する
- 撮影画像は保存しない
- サーバー側では一時ファイルとして扱い、解析後に削除する
- 本人確認、年齢判定、属性推定、個人識別には使わない
- 表情は断定せず「楽しそうに見える」程度の弱い情報として扱う
- キャラクターはカメラや解析の存在を言わない

## 保存する情報

`session_state.state_json.player_visible_reaction`

```json
{
  "mood": "amused",
  "engagement": "high",
  "confidence": 0.74,
  "short_note": "少し笑っているように見える。",
  "captured_at": "2026-05-02T01:00:00Z"
}
```

`mood`:

```text
amused
engaged
neutral
confused
uncomfortable
unknown
```

`engagement`:

```text
high
medium
low
unknown
```

## プロンプト反映

ライブチャットの返答生成には、以下のように弱い観察材料として渡す。

```text
Latest visible player reaction:
- mood=amused, engagement=high, confidence=0.74, note=少し笑っているように見える。, can_address_directly=true
```

反映ルール:

- 固定文字列は使わず、キャラクターの性格・口調・現在の話題に合わせてAIが自然な台詞を作る
- `confidence` が低い場合は直接触れない
- `mood` が `unknown` / `neutral` の場合は直接触れない
- 直近のキャラクター発話で表情・元気・疲れなどに触れていた場合は繰り返さない
- 直接触れない場合も、返答の温度感だけを少し調整してよい
- 「大丈夫？」「元気なさそう」などの気遣いを何度も繰り返さない
- カメラ、Webカメラ、解析、判定という言葉は出さない
- 断定せず「そう見えた」「そんな気がした」程度に留める

## UI

ライブチャット画面にカメラON/OFFボタンを追加する。

- OFF: カメラは使わない
- ON: ブラウザのカメラ許可を要求する
- ON中: キャラクター発話後に1枚だけキャプチャして送信
- 最後の反応を画面に表示する
- OFF時にカメラストリームを停止する

## API

`POST /api/v1/chat/sessions/<session_id>/player-reaction`

`multipart/form-data`

```text
file: image/jpeg
```

レスポンス:

```json
{
  "mood": "amused",
  "engagement": "high",
  "confidence": 0.74,
  "short_note": "少し笑っているように見える。",
  "captured_at": "..."
}
```
