# おでかけサービス 仕様/TODO

## 目的

チャットでもストーリーでもない、場所起点の短時間キャラクター体験を提供する。
ユーザーはワールドマップの施設を選び、キャラクターを誘い、2〜3ステップの選択肢イベントを進めて「今日の思い出」として保存する。

## 初期スコープ

- 左メニューに「おでかけ」を追加する。
- プロジェクト内のキャラクターとワールドマップ施設を選んで開始する。
- おでかけは独立したセッションとして保存する。
- 各ステップは本文、場所描写、キャラクターの反応、2〜3個の選択肢を持つ。
- 選択肢を押すと次のステップへ進む。
- 3ステップで完了し、思い出タイトル/要約/余韻メモを保存する。
- AIが使えない場合でもフォールバック文で体験が成立する。
- 施設画像とキャラ画像があれば画面に表示する。専用画像生成は次フェーズ。

## 画面の流れ

1. おでかけ画面を開く。
2. キャラクターと施設を選ぶ。
3. 「誘う」を押す。
4. イベント本文と選択肢が表示される。
5. 選択肢を押して進行する。
6. 完了後、思い出カードとして履歴に残る。

## API

- `GET /api/v1/projects/<project_id>/outings/options`
  - キャラクター、施設、最近のおでかけを返す。
- `GET /api/v1/projects/<project_id>/outings`
  - おでかけ履歴を返す。
- `POST /api/v1/projects/<project_id>/outings`
  - おでかけ開始。
- `GET /api/v1/outings/<outing_id>`
  - おでかけ詳細。
- `POST /api/v1/outings/<outing_id>/choose`
  - 選択肢を送って次へ進める。
