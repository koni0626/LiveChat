# キャラクター記憶機能

## 目的

キャラクターがユーザーとの会話の中で見せた「今後も残すと面白くなる一面」を、初期設定とは別の追加メモとして保存する。

このメモはキャラクター本体の設定ではなく、ユーザーごとの育成・関係性メモとして扱う。同じキャラクターでも、ユーザーAの会話で増えたメモはユーザーBには共有しない。

## 基本方針

- 初期設定の性格・口調・外見・NGルールは固定情報として優先する
- キャラクター記憶は `ユーザー × キャラクター` 単位で保存する
- AIは初期設定を書き換えず、自然に増えた一面だけを追加する
- 管理者が手動で追加・編集・無効化・削除できる
- プロンプト投入時は、そのセッション所有ユーザーに紐づくメモだけを渡す
- `user_id` が無い旧グローバルメモは共有漏れ防止のため読み込まない

## メモするもの

メモ対象は「これが残るとキャラクターが面白くなるか」で判断する。

- 好み: 好きな酒、食べ物、音楽、服、施設、趣味、作品傾向
- 癖: 照れると早口になる、話をそらす、強がる
- 価値観: 約束を重んじる、自由が好き、責任にこだわる
- 弱点/地雷: 苦手な話題、触れられると反応が変わること
- 関係性の芽: プレイヤーにだけ少し甘い、特定の呼び方を好む
- 伏線: 過去に何かありそう、特定の場所を避ける
- ネタ設定: BL好き、蒐集癖、妙なこだわり

## メモしないもの

- その場限りの雑談
- 一時的な感情だけの記録
- 初期設定と矛盾する推測
- プレイヤー固有の個人情報
- 他キャラクターに属する設定
- センシティブすぎる内容

## データ構造

テーブル名: `character_memory_note`

```text
id
user_id
character_id
category
note
source_type
source_ref
confidence
enabled
pinned
created_at
updated_at
```

`user_id` と `character_id` の組み合わせで、そのユーザーにとってのキャラクター更新メモを表す。

## AI抽出ルール

AIには会話後に以下を返させる。

```json
{
  "notes": [
    {
      "category": "preference",
      "note": "ノアはワインの香りに少し詳しい。",
      "confidence": 0.7
    }
  ]
}
```

ルール:

- 0件でもよい
- 初期設定を書き換えない
- 追加設定として自然なものだけ返す
- 既存メモとほぼ同じものは保存しない
- `confidence` が低すぎるものは保存しない
- 保存先はセッション所有ユーザーの `user_id` に限定する

## プロンプト反映

ライブチャットでは、キャラクター情報の近くに以下のように渡す。

```text
Character growth notes:
- preference: ワインが好きで、香りに少し詳しい。
- habit: 照れると話題を変える。
Use these as additive characterization. Do not override the base profile.
```

優先順位:

```text
初期設定 > NGルール > ユーザー別キャラクター記憶 > 直近会話
```

## UI

キャラクター設定画面に「キャラクターAIメモ」を表示する。

- メモ一覧
- 手動追加
- 編集
- 有効/無効切り替え
- ピン留め
- 削除

表示されるメモはログイン中ユーザーのものだけに限定する。
