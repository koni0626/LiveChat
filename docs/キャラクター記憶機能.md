# キャラクター記憶機能

## 目的

キャラクターが会話の中で見せた「今後も残すと面白くなる一面」を、初期設定とは別の追記型メモとして保存する。

初期設定の性格・口調・外見・NGルールは上書きしない。キャラクター記憶は、会話から増えた追加設定・嗜好・癖・伏線を補助情報として扱う。

## 基本方針

- 初期設定は固定情報として最優先する
- キャラクター記憶は追記型の追加設定として扱う
- AIは初期設定を修正せず、自然に増えた一面だけを候補化する
- 管理者は記憶を手動で追加・編集・無効化・削除できる
- プロンプト投入時は有効な記憶のみを短く渡す

## メモするもの

メモ対象は「これが残るとキャラクターが面白くなるか」で判断する。

- 嗜好: 好きな酒、食べ物、音楽、施設、趣味、作品傾向
- 癖: 照れると早口になる、話をそらす、強がる
- 価値観: 約束を重んじる、自由が好き、勝負にこだわる
- 弱点/地雷: 苦手な話題、触れられると反応が変わること
- 関係性の癖: プレイヤーにだけ少し甘い、特定の呼び方を好む
- 伏線: 過去に何かありそう、特定の場所を避ける
- ネタ設定: BL好き、収集癖、妙なこだわり

## メモしないもの

- その場限りの雑談
- 一時的な感情だけの記録
- 初期設定と矛盾する推測
- プレイヤー固有の個人情報
- 他キャラクターに属する設定
- センシティブすぎる内容

## データ構造

テーブル名: `character_memory_note`

```text
id
character_id
category
note
source_type
source_ref
confidence
enabled
pinned
created_at
updated_at
```

カテゴリ:

```text
preference
habit
value
weakness
relationship
foreshadowing
fun_fact
other
```

## AI抽出ルール

AIには会話後に以下を返させる。

```json
{
  "notes": [
    {
      "category": "preference",
      "note": "ノアはワインの銘柄に少し詳しい。",
      "confidence": 0.7
    }
  ]
}
```

ルール:

- 0件でもよい
- 初期設定を上書きしない
- 追加設定として自然なものだけ返す
- 1回の抽出では最大3件
- 既存メモとほぼ同じものは保存しない
- `confidence` が低すぎるものは保存しない

## プロンプト反映

ライブチャットでは、キャラクター情報の近くに以下のように渡す。

```text
Character growth notes:
- preference: ワインが好きで、銘柄に少し詳しい。
- habit: 照れると話題を変える。

Use these as additive characterization. Do not override the base profile.
```

優先順位:

```text
初期設定 > NGルール > キャラクター記憶 > 直近会話
```

## UI

キャラクター設定画面に「キャラ深掘りメモ」を追加する。

- メモ一覧
- 手動追加
- 編集
- 有効/無効切り替え
- ピン留め
- 削除

初期MVPでは一覧・追加・編集・有効/無効・削除を実装する。
