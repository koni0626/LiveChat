# セッション目的メモ機能

## 目的

ライブチャットの会話履歴ごとに、「この会話をもっと楽しく進めるにはどうするか」をDirectionAIがメモとして蓄積する。

キャラクター本体の目的や初期設定は書き換えない。ルーム/セッションに設定された初期目的も固定情報として扱う。

この機能で保存するのは、セッション単位の会話演出メモである。

## 方針

- 初期目的は固定情報として残す
- DirectionAIは初期目的を直接変更しない
- DirectionAIはセッション単位の目的メモを追加/更新/アーカイブする
- メモはそのチャット履歴の中だけで使う
- 会話全体の方針は `character_id = null`
- キャラ個別の方針は `character_id` に紐づける
- デバッグ確認のため、当面は会話ログの下に目的メモを表示する

## メモ対象

メモするもの:

- 会話が停滞しているときの次の打ち手
- 甘い反応が続きすぎたときの転換方針
- プレイヤーが反応した話題を次にどう広げるか
- キャラクターごとの次の役割や誘導方針
- 次に入れると面白い事件、謎、移動、関係性の揺れ
- そのセッションだけで有効な演出作戦

メモしないもの:

- キャラクターの恒久的な性格や好み
- 初期目的の書き換え
- ユーザーの個人情報
- 他セッションにも持ち越すべきキャラクター設定

## データ構造

テーブル名: `chat_session_objective_note`

```text
id
session_id
character_id nullable
title
note
priority
status
source_type
source_ref
confidence
created_at
updated_at
```

`status`:

```text
active
archived
```

`source_type`:

```text
direction_ai
manual
```

## DirectionAI 更新ルール

DirectionAIは会話後処理で、現在の目的メモと直近の会話を見て次を返す。

```json
{
  "notes": [
    {
      "action": "add",
      "scope": "session",
      "title": "甘さから事件へ切り替える",
      "note": "甘い反応が続いているので、次は小さな事件や謎を入れて会話に動きを作る。",
      "priority": 4,
      "confidence": 0.8
    }
  ]
}
```

`action`:

```text
add
update
archive
```

更新時は既存メモの `id` を指定する。

## プロンプト反映

ライブチャットでは、初期目的の近くに以下を渡す。

```text
Session objective notes:
- session: 甘い反応が続いているので、次は小さな事件を入れる。
- ノア: 照れ隠しから強気に誘導する。
Use these notes only for this chat history. Do not rewrite the initial session objective.
```

優先順位:

```text
初期目的 > 固定キャラクター設定 > キャラクター記憶 > セッション目的メモ > 直近会話
```

## UI

ライブチャットの会話ログ下に、デバッグ表示として「目的メモ」を表示する。

- 初期目的
- 有効なセッション目的メモ
- セッション全体/キャラ個別の区別
- 優先度
- DirectionAI由来かどうか

この表示は動作確認用で、うまく動いていることが分かれば後で削除または折りたたみ化する。
