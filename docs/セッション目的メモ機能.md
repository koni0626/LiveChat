# セッション目的メモ機能

## 目的

ライブチャットの会話履歴ごとに、「この会話をもっと楽しく進めるにはどうするか」をDirectionAIがメモとして蓄積する。

キャラクター本体の目的や初期設定は書き換えない。ルーム/セッションに設定された初期目的も固定情報として扱う。この機能で保存するのは、セッション単位の会話演出メモである。

## 基本方針

- 初期目的は固定情報として残す
- DirectionAIは初期目的を直接変更しない
- DirectionAIはセッション単位の目的メモを追加/更新/アーカイブする
- メモはそのチャット履歴の中だけで使う
- 会話全体の方針は `character_id = null`
- キャラ個別の方針は `character_id` に紐づける
- 目的更新時は、会話ログだけでなくお出かけ履歴・施設情報・ワールドニュース・Feed投稿も参照する
- DirectionAIは、目的だけでなく感情の出し方も短く方針化する
- 他キャラとのお出かけ、他キャラがニュース/Feedで注目されている状況は、嫉妬・寂しさ・対抗心・照れの材料にできる
- 成人キャラクター同士の会話では、露骨ではない範囲の色気、艶っぽい駆け引き、距離の近さ、含みのある冗談を作戦に含めてよい
- ただし、露骨な性行為、裸体描写、局部/胸部接触の詳細、強制、未成年/幼く見える性的表現は扱わず、大人の恋愛らしい緊張感へ変換する
- デバッグ確認のため、当面は会話ログの下に目的メモを表示する

## 参照する素材

DirectionAIは目的メモ更新時に以下を見る。

- 直近の会話ログ
- 現在の会話ディレクター出力
- 初期セッション目的
- 既存の目的メモ
- アクティブキャラクター情報
- セッション所有ユーザーの直近完了おでかけ
- ワールドマップ上の施設、地域、所有キャラクター
- 直近のワールドニュース、噂
- 直近の公開Feed投稿

これにより、以前のおでかけの思い出への自然な言及、施設への誘導、ニュースをきっかけにした事件、Feedで話題の投稿への反応などを、次の目的メモに反映できる。

## メモ対象

メモするもの:

- 会話が停滞しているときの次の打ち手
- 甘い反応が続きすぎたときの転換方針
- プレイヤーが反応した話題を次にどう広げるか
- キャラクターごとの次の役割や誘導方針
- 次に入れると面白い事件、噂、移動、関係性の揺れ
- 過去のおでかけを踏まえた自然な呼び戻しや思い出の再利用
- 施設・ニュース・Feedを使った具体的な会話フック
- 嫉妬、寂しさ、照れ、恥ずかしさ、怒り、喜びなどをどう会話に混ぜるか
- 大人の恋愛としての軽い色気、誘惑、含み、照れ、親密な距離感をどう混ぜるか
- そのセッションだけで有効な演出作戦

メモしないもの:

- キャラクターの恒久的な性格や好み
- 初期目的の書き換え
- ユーザーの個人情報
- 他セッションにも持ち越すべきキャラクター設定
- 毎ターン同じ照れ方や嫉妬を繰り返すだけの指示

## データ構造

テーブル名: `chat_session_objective_note`

```text
id
session_id
character_id nullable
title
note
priority
status
source_type
source_ref
confidence
created_at
updated_at
```

`status`:

```text
active
archived
```

`source_type`:

```text
direction_ai
manual
```

## DirectionAI 更新ルール

DirectionAIは会話後処理で、現在の目的メモと直近の会話、世界素材を見て次を返す。

```json
{
  "notes": [
    {
      "action": "add",
      "scope": "session",
      "title": "他キャラとのお出かけに少し焼きもちを混ぜる",
      "note": "プレイヤーが別キャラと水族館へ出かけた記憶があるため、次は露骨に責めず、少し拗ねた冗談や対抗する誘いで感情を出す。",
      "priority": 4,
      "confidence": 0.8
    }
  ]
}
```

`action`:

```text
add
update
archive
```

更新時は既存メモの `id` を指定する。

## プロンプト反映

ライブチャットでは、初期目的の近くに以下を渡す。

```text
Session objective notes:
- id=3 scope=session priority=4: 水族館のニュースを会話フックにする - 次は水族館の噂を使って会話を動かす。
Use these notes only for this chat history. Do not rewrite the initial session objective.
```

優先順位:

```text
初期目的 > 固定キャラクター設定 > キャラクター記憶 > セッション目的メモ > 直近会話
```

## UI

ライブチャットの会話ログ下に、デバッグ表示として「目的メモ」を表示する。

- 初期目的
- 有効なセッション目的メモ
- セッション全体/キャラ個別の区別
- 優先度
- DirectionAI由来かどうか

この表示は動作確認用で、うまく動いていることが分かれば後で削除または折りたたみ化する。
