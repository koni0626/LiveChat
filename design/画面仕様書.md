# AIノベルゲームツクール 画面設計書

## 1. 目的

本書は、AIノベルゲームツクールの主要画面について、役割、構成、表示項目、操作内容、画面遷移を定義する。

本ツールは、ユーザーが生成AIを利用してノベルゲームを制作するための制作環境であり、単なるプレイヤー向けUIではなく、**制作者向けの編集UI** を中心に設計する。

## 3. 画面遷移概要

```text
ログイン
  ├─ 新規登録
  │    ↓
  │  ダッシュボード
  ↓
ダッシュボード
  ├─ 作品一覧
  │    ├─ 作品新規作成
  │    └─ 作品編集トップ
  │          ├─ キャラクター一覧 → キャラクター編集
  │          ├─ 世界観設定
  │          ├─ ストーリー骨子設定
  │          ├─ シーン一覧・編集
  │          │      ├─ シーン生成結果確認
  │          │      └─ 画像生成確認
  │          ├─ プレビュー
  │          └─ エクスポート
  └─ 設定
```

## 4.8 キャラクター編集画面

### 目的

* キャラクターの人格と外見を定義する

### レイアウト

* 左: 基本情報
* 中央: 性格・口調・役割設定
* 右: 基準画像、画像生成固定設定

### 入力項目

#### 基本情報

* 名前
* 役割
* 年齢感
* 一人称
* 二人称

#### 性格・口調

* 性格説明
* 話し方
* 口調サンプル
* NG表現
* 禁止設定

#### 外見

* 基準画像アップロード
* 基準画像プレビュー
* 髪型
* 目元
* 耳の種類
* 装飾
* 服装傾向
* 雰囲気キーワード

#### 画像固定ルール

* 顔を極力固定
* human ears を明示
* lowモード標準
* 衣装固定 / 可変
* 基準画像を参照画像として使う
* 必要に応じて image-to-image / reference image ベースで生成する

### 操作

* 保存
* AI補完
* 画像テスト生成

### 備考

* キャラクターの安定性を最重要視するため、この画面はMVPでも重点実装対象
* 基準画像がある場合は、画像生成時に参照画像として利用し、顔・髪型・装飾の再現性を上げる

## 6. 画面別バリデーション方針

### 共通

* 必須項目未入力時は保存不可
* エラー箇所を画面内で明示
* AI生成失敗時は再試行導線を表示

### 個別

* キャラクター編集: 基準画像未設定でも保存可。ただし警告表示
* 世界観設定: 固有名詞辞書が空でも保存可
* シーン編集: 本文ゼロでもAI生成可能

## 8. UX設計方針

### 8.1 基本方針

* 生成AIの不安定さを前提に、**すぐ修正できるUI** にする
* 「一発で完成」より「生成→直す→採用」を高速に回せることを重視
* 画像生成コストを意識し、無駄な再生成を抑える導線にする

### 8.2 重視する体験

* キャラクターを作るのが楽しい
* シーンが増えていく感触がある
* すぐにプレビューで確認できる
* 生成結果を怖がらず触れる

### 8.3 避けたい体験

* 設定画面が複雑すぎて何も始められない
* 画像生成が重いのに結果が見えづらい
* 編集画面とプレビューが離れすぎて確認しづらい

## 10. まとめ

本ツールの画面設計は、プレイヤー向けUIではなく、**制作者が生成AIと共同でノベルゲームを組み立てるための制作UI** を中心に構成する。

特に重要な画面は、

* キャラクター編集画面
* シーン一覧・編集画面
* 画像生成確認画面
* プレビュー画面

であり、これらがMVPの中核となる。
