# LiveChat 選択機能 仕様書

## 1. 目的

ライブチャット中のキャラクター発話から、場面転換や行動選択につながる候補を自動抽出し、ユーザーがボタンで選べるようにする。

例:

```text
キャラクター:
……うん、もももがそう言うなら、今日は“仕事を考える時間”まで、海に連れていってもらおうかな。
```

この発話から `海へ行く` という選択肢を表示する。
ユーザーが選択肢を押すと、通常の返答テキストではなく「海へ移動した場面」として画像生成を行い、必要に応じてキャラクターが一言反応する。

## 2. 用語

| 用語 | 意味 |
| --- | --- |
| 選択肢 | ユーザーが次の行動・移動・場面転換を選ぶためのボタン |
| 選択候補 | AIまたはルールで抽出された、まだ押されていない選択肢 |
| 場面転換 | ユーザーの選択によって、背景・場所・状況が変わること |
| 選択実行 | 選択肢ボタンを押し、画像生成と会話反応を行う処理 |
| 選択肢表示レイヤ | ライブチャット画像の中央付近に表示するボタンUI |

## 3. 基本方針

- 選択肢はキャラクターの最新発話をもとに自動生成する。
- 選択肢は必ず表示するものではない。場面転換や行動候補がある場合だけ表示する。
- 選択肢を押した場合、ユーザーの通常メッセージ送信とは別扱いにする。
- 選択肢を押した後は、選択肢を消す。
- 選択肢を押した結果、画像を自動生成する。
- 画像生成後、キャラクターは生成された場面を前提に短く反応する。
- 選択肢はライブチャットセッション単位で管理する。
- DB追加はMVPでは行わず、`session_state.state_json` に一時保存する。

## 4. ユーザーストーリー

1. ユーザーがキャラクターと会話する。
2. キャラクターが「海へ行きたい」「山もいい」など、移動・行動候補を含む発話をする。
3. システムが発話を解析し、選択肢候補を作成する。
4. ライブチャット画像の中央付近に `海へ行く` `山へ行く` のようなボタンが表示される。
5. ユーザーがボタンを押す。
6. システムは選択内容を場面転換指示として扱う。
7. 選択内容に合う画像を生成する。
8. 生成画像がメイン画像に反映される。
9. キャラクターがその場面に合う一言を返す。

## 5. 対象となる選択肢

### 5-1. 表示する候補

- 場所移動: `海へ行く`, `山へ行く`, `店に入る`, `部屋へ戻る`
- 行動: `手をつなぐ`, `隣に座る`, `プレゼントを渡す`
- 見る対象: `夜景を見る`, `クルーザーを見る`
- 話題選択: `仕事の話をする`, `恋愛の話をする`

### 5-2. 表示しない候補

- 単なる相づち: `うん`, `そうだね`
- キャラクターの独白だけで、ユーザーの選択余地がないもの
- 画像や場面に反映しづらい抽象的すぎるもの
- キャラクターが明確に拒否している行動
- キャラクター設定やNGルールに反する行動

## 6. 選択肢抽出仕様

### 6-1. 入力

選択肢抽出AIには以下を渡す。

- 最新のキャラクター発話
- 直近の会話ログ
- 現在の場所・背景・状況
- セッションの目的
- キャラクター設定
- NGルール
- 現在表示中の画像解析結果があれば、その内容

### 6-2. 出力

AIはJSONで返す。

```json
{
  "should_show_choices": true,
  "choices": [
    {
      "label": "海へ行く",
      "intent": "scene_transition",
      "scene_instruction": "キャラクターと一緒に海へ移動する。開放感のある海辺の場面を生成する。",
      "image_prompt_hint": "海辺、波、夕方、キャラクターが楽しそうにこちらを見る",
      "reply_hint": "海に着いた後、キャラクターが嬉しそうに一言話す"
    }
  ]
}
```

### 6-3. 選択肢数

- 0から3件まで。
- MVPでは最大2件を推奨する。
- 3件以上ある場合、AIが魅力的なものを優先して絞る。

### 6-4. ラベル

- ボタンラベルは短くする。
- 推奨形式: `海へ行く`, `山へ行く`, `夜景を見る`
- 長文は禁止。
- 句読点は基本的に使わない。

## 7. 保存仕様

MVPでは `session_state.state_json.scene_choices` に保存する。

```json
{
  "scene_choices": {
    "source_message_id": 123,
    "created_at": "2026-04-26T12:00:00",
    "choices": [
      {
        "id": "choice_1",
        "label": "海へ行く",
        "intent": "scene_transition",
        "scene_instruction": "キャラクターと一緒に海へ移動する。開放感のある海辺の場面を生成する。",
        "image_prompt_hint": "海辺、波、夕方、キャラクターが楽しそうにこちらを見る",
        "reply_hint": "海に着いた後、キャラクターが嬉しそうに一言話す"
      }
    ]
  }
}
```

選択肢実行後は `scene_choices` を削除または `consumed: true` にする。

## 8. API仕様

### 8-1. コンテキスト取得

既存API:

```http
GET /api/v1/chat/sessions/{session_id}
```

レスポンスの `state.state_json.scene_choices` をフロントが参照する。

### 8-2. 選択肢実行

新規API:

```http
POST /api/v1/chat/sessions/{session_id}/choices/{choice_id}/execute
```

リクエスト:

```json
{
  "choice_id": "choice_1"
}
```

処理:

1. `session_state.state_json.scene_choices` から対象選択肢を取得する。
2. 対象選択肢が存在しない場合は404。
3. `scene_instruction` を場面転換指示として扱う。
4. 画像生成用プロンプトを作る。
5. 選択中の衣装画像を参照画像として使う。
6. 画像を生成し、`session_image.image_type = directed_scene` として保存する。
7. 生成画像を選択状態にする。
8. キャラクターの短い反応メッセージを作成する。
9. `scene_choices` を消す。
10. 最新コンテキストを返す。

レスポンス:

```json
{
  "selected_choice": {
    "id": "choice_1",
    "label": "海へ行く"
  },
  "generated_image": {
    "id": 456,
    "asset": {
      "media_url": "/media/..."
    }
  },
  "messages": [],
  "state": {}
}
```

## 9. 画面仕様

### 9-1. 表示位置

ライブチャット画面のメイン画像上、または画像直下の中央に表示する。

推奨:

```text
[画像]
      [海へ行く] [山へ行く]
[テキストボックス]
```

画像に完全に重ねると邪魔になる可能性があるため、MVPでは画像の下、テキストボックスの上に表示する。

### 9-2. 表示条件

- `scene_choices.choices` が1件以上ある。
- まだ選択肢が消費されていない。
- 画像生成中・返信生成中ではない。

### 9-3. 操作

- ボタンを押すと即実行する。
- 実行中はボタンを無効化し、`移動中...` または `場面を作成中...` と表示する。
- 実行完了後、選択肢UIは消える。

### 9-4. デザイン

- サイバーネオンデザインに合わせる。
- ボタンは中央寄せ。
- 複数候補は横並び。スマホでは縦並び。
- ボタン文言は大きめにし、押せることが分かる見た目にする。

## 10. 会話生成との関係

### 10-1. 通常会話

通常のユーザーメッセージ送信後:

1. ユーザーメッセージ保存
2. キャラクター返信生成
3. キャラクター返信保存
4. キャラクター返信から選択肢候補を抽出
5. `scene_choices` に保存
6. フロントに表示

### 10-2. 選択実行

選択肢ボタン押下時:

1. 選択肢を取得
2. 場面転換画像を生成
3. 画像を表示
4. キャラクターの反応を生成
5. 選択肢を消す

通常の `話を進めて` とは別の処理にする。

## 11. 画像生成仕様

選択肢実行時の画像生成では、以下を重視する。

- 選択肢ラベル
- `scene_instruction`
- `image_prompt_hint`
- 直近の会話文脈
- 現在の衣装ルームで選択中の基準画像
- キャラクター設定の画風
- プレイヤー1人称視点
- 画像内にセリフ・字幕・文字を入れない

例:

```text
ユーザーが「海へ行く」を選択した。
直近の会話では、キャラクターが仕事を忘れるために海へ行きたいと話している。
プレイヤー1人称視点。
キャラクターだけを画面に表示し、プレイヤーは描かない。
海辺に到着した場面。
選択中の衣装画像と同じ顔、髪型、体型、衣装を維持する。
画像内にセリフ、字幕、文字、ロゴを入れない。
```

## 12. キャラクター反応仕様

画像生成後、キャラクターは短く反応する。

例:

```text
……ふふ、ほんとに海まで来ちゃったね。ねえ、少しだけ隣にいてくれる？
```

ルール:

- 長文にしない。
- 画像の場面を前提に話す。
- キャラクターの性格・話し方を反映する。
- 説明口調にしない。
- 「案内する」は使わない。

## 13. エラー仕様

### 13-1. 選択肢が存在しない

```json
{
  "message": "選択肢が見つかりません。"
}
```

### 13-2. 画像生成失敗

- 選択肢は消さない。
- ユーザーにエラーを表示する。
- 再実行できるようにする。

### 13-3. 反応メッセージ生成失敗

- 画像生成が成功していれば画像は表示する。
- キャラクター反応だけ短いフォールバックにする。

```text
……来ちゃったね。
```

## 14. 権限

選択肢実行は、対象ライブチャットセッションを閲覧・操作できるユーザーのみ可能。

| 操作 | superuser | project_user | user |
| --- | --- | --- | --- |
| 自分のセッションで選択肢を見る | 可 | 可 | 可 |
| 自分のセッションで選択肢を実行 | 可 | 可 | 可 |
| 他人のユーザーセッションで選択肢を実行 | 可 | 不可 | 不可 |

## 15. MVP実装手順

1. 選択肢抽出用のテキストAI関数を追加する。
2. キャラクター返信生成後に選択肢抽出を呼ぶ。
3. `session_state.state_json.scene_choices` に保存する。
4. ライブチャット画面で `scene_choices` を表示する。
5. 選択肢実行APIを追加する。
6. 選択肢実行時に画像生成を行う。
7. 生成画像をメイン画像に反映する。
8. キャラクターの短い反応を保存する。
9. 選択肢を消す。

## 16. MVP完了条件

- キャラクターが場所候補を含む発話をした場合、選択肢ボタンが表示される。
- `海へ行く` `山へ行く` のような複数選択肢を表示できる。
- 選択肢ボタンを押すと、対応する場面画像が生成される。
- 生成画像がメイン画像に反映される。
- 選択肢実行後、キャラクターが場面に合った一言を返す。
- 選択肢実行後、選択肢ボタンは消える。
- 画像生成失敗時、選択肢は残り再実行できる。

## 17. 将来拡張

- 選択肢履歴を専用テーブルで保存する。
- 選択肢ごとの成功率や恋愛度変化を記録する。
- 選択肢に必要条件を付ける。
- 選択肢に消費アイテムや贈り物を紐づける。
- 選択肢から分岐イベントやお手紙発生につなげる。
- プロジェクトユーザーがルームごとに選択肢抽出の強さを設定できるようにする。
