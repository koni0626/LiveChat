# LiveChat 衣装ルーム機能 仕様書

## 1. 目的

ライブチャット中のイベント画像と、キャラクターの見た目を固定するための基準画像を分離する。

現状はライブチャット右側の生成画像一覧から基準画像を選択しているため、背景・ポーズ・構図を含むイベント画像が次回画像生成の参照画像になり、顔や衣装、背景が不安定になりやすい。

衣装ルームでは、キャラクター単体の衣装・外見参照画像を管理し、選択中の衣装画像をライブチャット画像生成の参照画像として使う。

## 2. 用語

| 用語 | 意味 |
| --- | --- |
| イベント画像 | ライブチャット画面中央に表示する、会話や場面に応じた一枚絵 |
| 衣装ルーム | ライブチャット内でキャラクターの衣装基準画像を作成・選択する領域 |
| 衣装画像 | キャラクター単体の参照画像。背景やシーン演出ではなく、顔・髪・衣装・体型の固定に使う |
| 初期衣装 | キャラクター設定画面の `base_asset_id` をもとにした最初の衣装画像 |
| 選択中衣装 | 以後のライブチャット画像生成で reference image として使う衣装画像 |

## 3. 基本方針

- ライブチャットのイベント画像一覧から「基準画像を選ぶ」UIは廃止する。
- イベント画像は会話・場面・背景を楽しむための画像として扱う。
- 衣装画像はキャラクターの見た目を固定するための参照画像として扱う。
- ライブチャット画像生成時は、選択中衣装画像を reference image として使う。
- 衣装画像はセッション単位で管理する。
- 同じルームでもユーザーごとに衣装ルームは独立する。

## 4. ユーザーストーリー

1. ユーザーがライブチャットを開始する。
2. 衣装ルームには、キャラクター設定の基準画像が初期衣装として表示される。
3. ユーザーが「清楚系のワンピースに着替える」のように衣装変更プロンプトを入力する。
4. 選択中衣装画像を参照画像として、新しい衣装画像を生成する。
5. 生成された衣装画像が衣装ルームに追加され、自動的に選択中衣装になる。
6. 以後のライブチャット画像生成では、その衣装画像を参照してイベント画像を生成する。
7. ユーザーは過去に作成した衣装画像を選択し直せる。

## 5. 画面仕様

### 5-1. ライブチャット画面

ライブチャット画面に「衣装ルーム」セクションを追加する。

配置案:

```text
[メイン画像]

[画像生成] [テキストボックス表示/非表示]

[衣装ルーム ▼]
  現在の衣装プレビュー
  衣装変更プロンプト
  [衣装を生成]
  衣装サムネイル一覧

[ログ]
```

### 5-2. 衣装ルーム表示項目

- 現在選択中衣装の大きめプレビュー
- 衣装変更プロンプト入力欄
- 衣装生成ボタン
- 衣装画像サムネイル一覧
- 選択状態
- 作成日時
- 生成プロンプト確認
- 削除ボタン

### 5-3. 衣装画像サムネイル

- ラジオボタンまたはカード選択で1枚だけ選択できる。
- 選択中カードはネオン枠などで明確に表示する。
- 最新生成画像は自動的に選択中にする。
- 初期衣装は削除不可にするか、削除してもキャラクター基準画像から復元できるようにする。

### 5-4. 既存画像一覧から削除するUI

ライブチャットのイベント画像一覧から以下を削除する。

- 基準画像ラジオボタン
- 基準画像として選択する操作
- `is_reference` をユーザーに直接意識させる表示

イベント画像一覧には以下だけを残す。

- 表示画像の選択
- 画像拡大
- 画像削除
- プロンプト確認

## 6. 画像生成仕様

### 6-1. 衣装画像生成

入力:

- 現在選択中衣装画像
- キャラクター設定
- キャラクター画風
- ユーザーの衣装変更プロンプト

出力:

- キャラクター単体の衣装参照画像

生成方針:

- 1人だけ描く。
- 顔、髪型、体型、キャラクター性を保つ。
- 背景はシンプルにする。
- イベントCGのような複雑な背景・構図にはしない。
- 衣装が分かるように全身または膝上程度を優先する。
- 文字、字幕、吹き出し、ロゴ、透かしは入れない。
- プレイヤーは描かない。

プロンプト例:

```text
現在のキャラクター基準画像を元に、同一人物として衣装だけを変更する。
清楚系の白いワンピースに着替える。
顔、髪型、体型、キャラクター性、画風は維持する。
キャラクター単体、全身、シンプル背景。
画像内に文字、字幕、吹き出し、ロゴ、透かしを入れない。
```

### 6-2. ライブチャット画像生成

ライブチャットのイベント画像生成では、参照画像として以下を使う。

優先順位:

1. セッションの選択中衣装画像
2. キャラクター設定の `base_asset_id`

イベント画像生成では、背景やシーンは会話状態・画像解析結果・場面転換情報から作る。

衣装画像は、顔・衣装・見た目を固定するためだけに使う。

### 6-3. 場面転換との関係

場面転換時に自動生成されるイベント画像も、選択中衣装画像を参照する。

例:

```text
入力: そして僕たちは店の外に出た。
現在の衣装: 白いワンピース
生成結果: 白いワンピース姿のキャラクターが店の外の歩道にいるイベント画像
```

### 6-4. 贈り物画像との関係

贈り物リアクション画像でも、選択中衣装画像を参照する。

例:

```text
プレイヤーがぬいぐるみ画像を贈る。
現在の衣装画像を参照し、その衣装のままぬいぐるみを抱くイベント画像を生成する。
```

## 7. DB設計

### 7-1. MVP方針

MVPでは既存の `session_image` を流用する。

`image_type` に以下を追加する。

| image_type | 用途 |
| --- | --- |
| `live_scene` | 通常のライブチャットイベント画像 |
| `directed_scene` | 場面転換で生成されたイベント画像 |
| `gift_reaction` | 贈り物リアクション画像 |
| `costume_reference` | 衣装ルームで管理する衣装画像 |
| `costume_initial` | キャラクター基準画像から作る初期衣装 |

### 7-2. 選択状態

衣装画像の選択状態は以下で表す。

```text
session_image.image_type in ('costume_reference', 'costume_initial')
session_image.is_selected = 1
```

ただし、イベント画像側の `is_selected` と意味が衝突する可能性があるため、実装時に以下のどちらかを選ぶ。

案A:

- `session_image.is_selected` を画像タイプごとに独立して扱う。
- `live_scene` の選択と `costume_reference` の選択は別グループとして制御する。

案B:

- 新規カラム `selection_group` を追加する。
- `display` と `costume` を分ける。

MVPでは案Aを採用する。

### 7-3. 将来の専用テーブル案

衣装ルームが複雑になった場合は、専用テーブルを追加する。

```text
session_costume_reference
- id
- session_id
- character_id
- asset_id
- source_session_image_id
- prompt_text
- is_selected
- is_initial
- created_at
- updated_at
- deleted_at
```

MVPでは追加しない。

## 8. API仕様

### 8-1. 衣装一覧取得

```http
GET /api/v1/chat/sessions/{session_id}/costumes
```

返却:

```json
{
  "data": [
    {
      "id": 1,
      "asset_id": 10,
      "image_type": "costume_reference",
      "prompt_text": "清楚系のワンピース",
      "is_selected": true,
      "asset": {
        "media_url": "/media/..."
      }
    }
  ]
}
```

### 8-2. 衣装生成

```http
POST /api/v1/chat/sessions/{session_id}/costumes/generate
```

リクエスト:

```json
{
  "prompt_text": "清楚系の白いワンピースに着替える",
  "size": "1024x1536",
  "quality": "medium"
}
```

処理:

1. 現在選択中衣装を取得する。
2. 未選択の場合はキャラクター基準画像を使う。
3. image-to-image で衣装画像を生成する。
4. `session_image.image_type = costume_reference` として保存する。
5. 生成画像を選択中衣装にする。

### 8-3. 衣装選択

```http
POST /api/v1/chat/sessions/{session_id}/costumes/{session_image_id}/select
```

処理:

1. 対象画像が `costume_reference` または `costume_initial` であることを確認する。
2. 同一セッション内の衣装画像の `is_selected` を解除する。
3. 対象画像を選択状態にする。

### 8-4. 衣装削除

```http
DELETE /api/v1/chat/sessions/{session_id}/costumes/{session_image_id}
```

制約:

- `costume_initial` は削除不可、または削除時に確認を出す。
- 選択中衣装を削除した場合は、初期衣装へ戻す。

## 9. 初期衣装作成

ライブチャットセッション作成時、キャラクターに `base_asset_id` があれば、衣装ルームに初期衣装として登録する。

登録方法:

- 新しいassetは作らない。
- `session_image` に `asset_id = character.base_asset_id` で `costume_initial` 行を作る。
- `is_selected = 1` にする。

キャラクター基準画像が未設定の場合:

- 衣装ルームには空状態を表示する。
- 衣装生成ボタンは無効、または「先にキャラクター基準画像を登録してください」と表示する。

## 10. 既存機能への影響

### 10-1. 右サムネイルの基準画像選択

廃止する。

既存の `is_reference` は互換性のためDBには残すが、ライブチャットUIでは使わない。

### 10-2. 画像生成参照元

現在:

```text
セッション画像の is_reference = 1
```

変更後:

```text
選択中の costume_reference / costume_initial
```

### 10-3. キャラクター設定の基準画像

キャラクター設定の基準画像は、以下に使う。

- 新規セッションの初期衣装
- 衣装ルームが未作成の場合のフォールバック
- キャラクター一覧サムネイル

### 10-4. 画像理解機能

ライブチャットのイベント画像生成後は、既存の画像理解処理で表示中画像を解析する。

衣装画像生成時は、画像理解を必須にはしない。

必要であれば衣装画像に対しても以下を保存できる。

```json
{
  "costume_summary": "白いワンピース",
  "changed_parts": ["服", "靴"],
  "kept_parts": ["顔", "髪型", "体型"]
}
```

## 11. 権限

ライブチャットセッション所有者のみ、自分の衣装ルームを操作できる。

| 操作 | superuser | project_user | user |
| --- | --- | --- | --- |
| 自分の衣装ルーム閲覧 | 可 | 可 | 可 |
| 自分の衣装生成 | 可 | 可 | 可 |
| 自分の衣装選択 | 可 | 可 | 可 |
| 他ユーザーの衣装ルーム閲覧 | 可 | 原則不可 | 不可 |
| 他ユーザーの衣装変更 | 可 | 不可 | 不可 |

## 13. 実装順

1. `session_image` を使って衣装画像一覧を取得するServiceを追加する。
2. セッション作成時に `costume_initial` を作成する。
3. 衣装一覧APIを追加する。
4. 衣装生成APIを追加する。
5. 衣装選択APIを追加する。
6. ライブチャット画像生成時の参照画像取得先を衣装ルームに変更する。
7. ライブチャットUIに衣装ルームを追加する。
8. 既存イベント画像一覧から基準画像選択UIを削除する。
9. 既存 `is_reference` 依存を段階的に削除する。

## 14. 未決事項

- 初期衣装を削除不可にするか、削除可能にして復元ボタンを置くか。
- 衣装画像を全身固定にするか、膝上も許可するか。
- 衣装生成のサイズを `1024x1536` 固定にするか、ユーザー設定に従うか。
- 衣装画像にもVision解析を行い、衣装説明を保存するか。
- 複数キャラクター対応を将来どう扱うか。

## 15. MVP完了条件

- ライブチャット画面に衣装ルームが表示される。
- キャラクター基準画像が初期衣装として表示される。
- 衣装変更プロンプトから新しい衣装画像を生成できる。
- 衣装画像を1枚選択できる。
- ライブチャット画像生成時、選択中衣装画像が参照画像として使われる。
- イベント画像一覧から基準画像選択UIが消えている。
