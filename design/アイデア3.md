昔のCGIゲームっぽさって、要するに「非同期」「ログが残る」「少しずつ育つ」「他人の行動が世界に痕跡として残る」なので、AI画像生成と合わせるとかなり独特の味が出ます。

たとえばこんな方向が面白そうです。

**1. ラプラスシティ住民シミュレーション**
プレイヤーは都市の住民、観測者、労働者、配信者、異端者などになって、1日数回だけ行動を選ぶ。

行動例:
- 仕事に行く
- 路地裏を探索する
- キャラに会いに行く
- 噂を流す
- 店を開く
- 事件を調べる
- 誰かに贈り物をする

結果として、CGイラスト付きの「今日の出来事」が生成される。  
昔の箱庭CGIゲームっぽく、全員の行動結果がワールドニュースや街ログに流れる。

かなり今のアプリ資産と相性がいいです。

**2. AI美少女酒場 / サロン経営CGI**
プレイヤーはラプラスシティの小さな店を持つ。キャラたちが日替わりで来店する。

できること:
- 店の内装を変える
- メニューを作る
- キャラを招待する
- 常連との関係を育てる
- 今日のイベントCGを生成する
- 店の噂がワールドニュースに流れる

昔の「箱庭経営」「街の掲示板」「ランキング」感が出せます。  
画像生成は、店内CG、来店イベントCG、宣伝ポスターに使える。

**3. ラプラス学園 / 都市学院もの**
1日1ターン制の学園CGI。

行動:
- 授業
- 部活
- 放課後イベント
- キャラと会話
- 噂を調べる
- 学園祭準備
- 怪異調査

各ターンでイベントCGが出る。  
他プレイヤーの行動で、学校新聞や噂掲示板が更新される。  
「今日は誰と誰が屋上で話していた」みたいなCGIゲームの懐かしさが出ます。

**4. 迷宮探索＋キャラ交流**
昔のブラウザRPGっぽく、プレイヤーは探索者。  
1日数回だけダンジョンに潜れる。

面白い点:
- 探索結果が毎回CGイラスト化される
- 拾った素材で衣装や部屋を作る
- キャラとパーティを組む
- 戦闘は軽め、イベント重視
- 他プレイヤーが見つけた場所が街に追加される

これはゲーム性を作りやすいです。  
画像生成は「発見した部屋」「ボス」「宝物」「キャラの戦闘スチル」に効く。

**5. 推し活・配信者シミュレーション**
ミウみたいなキャラを中心にすると強いです。

プレイヤーはファン、配信者、運営、スポンサーなどになる。  
毎日、配信イベント、投げ銭、炎上、コラボ、限定衣装、握手会みたいなイベントが起きる。

生成画像:
- 今日の配信サムネ
- 限定チェキ
- 炎上ニュース画像
- ライブステージCG
- ファン向け特典画像

これは風刺と中毒性が出ます。  
「無料の孤独より、有料のぬくもり」系の世界観と相性抜群。

**6. 街の噂で物語が進むゲーム**
プレイヤーは直接大事件を解決しない。  
毎日「誰に会うか」「どこへ行くか」「何を噂するか」を選ぶだけ。

すると世界ニュース、キャラ関係、ノベル、画像が少しずつ変わる。

CGIゲームっぽい要素:
- 掲示板
- 住民ログ
- 噂一覧
- 人気キャラランキング
- 未解決事件
- 週ごとの都市イベント
- プレイヤーごとの日記

これは今のワールドニュース機能を中核にできそうです。

個人的に一番おすすめは、

**「ラプラスシティ1日1ターン生活ゲーム」**

です。

プレイヤーは1日に数回だけ行動する。  
行動結果はAIが短いイベント文＋CGイラストで生成。  
結果がキャラ記憶、ワールドニュース、ノベルの種、街ログに蓄積される。

画面構成はこんな感じ。

- 今日の都市ニュース
- 自分の行動ポイント
- 行ける場所
- 会えるキャラ
- 今日のおすすめ行動
- 最近の他プレイヤーの噂
- 自分の日記
- 生成されたイベントCGアルバム

ゲーム性は軽いけど、世界が育つ。  
昔のCGIゲームの「毎日ちょっと見る」感じと、今のAI生成の強みが噛み合います。

方向性としては、最初は戦闘や複雑なステータスより、

**行動 → イベント → CG → ニュース化 → キャラ記憶に反映**

このループだけ作るのがよさそうです。  
これだけでかなり遊べます。

そう、かなりFeedに近いです。  
ただFeedが「キャラが投稿する世界の表面」だとすると、1日1回行動ゲームは **プレイヤーの選択で世界に1つ痕跡を残す仕組み** にできます。

方向性は大きく2つあります。

**A. 全キャラクターが勝手に動く観察型**
毎日、登録キャラ全員または一部が自動で行動する。

例:
- ノアが観測塔で変なログを見つけた
- ドルが新しい投資商品を売り出した
- ミウが深夜配信で孤独指数を上げた
- ハタラケ姫が労働キャンペーンを開始した

それがFeedやワールドニュースに流れる。  
プレイヤーは「観測する」「反応する」「介入する」くらい。

これは世界が生きて見えるけど、ゲーム性は弱めです。  
どちらかというと **都市観測シミュレーター**。

**B. プレイヤーの1行動をスコア評価するゲーム型**
プレイヤーが1日1回、誰と何をするか選ぶ。

例:
- ノアと観測塔に行く
- ミウの配信を手伝う
- ドルの商談に乗る
- ラプラスに都市ログを見せてもらう

結果としてスコアが変わる。

スコア例:
- 親密度
- 信頼
- 噂度
- 承認
- 疲労
- 所持金
- 観測値
- 都市影響度
- 危険度
- 非観測度

結果は短いイベント文＋CG＋Feed/ニュース化。  
これはちゃんとゲームになります。

僕のおすすめは **A+Bの混合** です。

毎日、世界側ではキャラたちが勝手に少し動く。  
その上で、プレイヤーは1日1回だけ「誰に介入するか」を選ぶ。

たとえば:

1. 朝に「今日の都市ログ」が生成される  
   - キャラ3人くらいが勝手に動いた結果がFeed/ニュースに出る

2. プレイヤーは1日1回だけ行動する  
   - 誰に会う
   - どこへ行く
   - 何を調べる
   - 何を贈る
   - 何を投稿する

3. 結果が出る  
   - イベント文章
   - CGイラスト
   - スコア変化
   - キャラ記憶
   - Feed投稿
   - ワールドニュース

スコアは入れた方がいいです。  
でも最初から数値ゲームにしすぎると作業感が出るので、見せ方は控えめが良さそう。

たとえば内部ではこう持つ:

- `親密度 +3`
- `都市影響 +1`
- `疲労 +2`
- `噂度 +4`
- `危険度 +1`
- `承認 +5`

画面には全部出さずに、

> ミウとの距離が少し近づいた。  
> 都市の噂が少しざわついた。  
> あなたは少し疲れた。

くらいで見せる。

昔のCGIゲーム感を出すなら、スコア名はもっと世界観に寄せたいです。

ラプラスシティなら:

- **観測値**: 都市にどれだけ見られているか
- **非観測値**: 誰にも見られない自由度
- **承認流通量**: いいね、噂、注目
- **関係深度**: キャラとの距離
- **労働圧**: 働け圧
- **孤独濃度**: ミウ系イベントに影響
- **信用残高**: ドル系イベントに影響
- **違反兆候**: 都市システムに目をつけられる度合い

良い基本ループはこれです。

**今日の都市ログ**
キャラたちが勝手に動く。世界がざわつく。

**今日の一手**
プレイヤーが1回だけ選ぶ。

**結果**
文章、CG、スコア、記憶、ニュースが生成される。

**翌日**
前日の結果を踏まえて、選択肢とキャラの動きが変わる。

個人的には、初期実装はこれで十分です。

- 毎日3件、キャラ自動ログを生成
- プレイヤーは1日1回行動
- 行動結果にCG1枚
- 内部スコアは5種類くらい
- Feedとワールドニュースに反映
- キャラ記憶にも要約を保存

これなら「Feedの延長」に見えつつ、ちゃんとゲームになります。
