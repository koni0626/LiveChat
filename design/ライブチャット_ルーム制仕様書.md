# ライブチャット ルーム制仕様書

## 1. 目的
ライブチャットの設定項目を、プロジェクトユーザーが管理する「ルーム」と、一般ユーザーが実際に会話する「セッション」に分離する。

現状は、セッション名、会話の目的、話すキャラクター、プレイヤー名が同じ画面・同じセッション設定に混在している。公開運用では、一般ユーザーが会話の目的や相手キャラクターを自由に変更できると、プロジェクトユーザーが設計した体験を維持できない。

そのため、会話設計はルームに固定し、ユーザーごとの会話履歴はセッションとして個別に保存する。

## 2. 用語
| 用語 | 説明 |
|---|---|
| プロジェクト | 世界観、キャラクター、ライブチャットルームをまとめる親コンテンツ |
| ルーム | プロジェクトユーザーが作成する会話テンプレート。会話の目的、話すキャラクター、公開状態を持つ |
| セッション | ユーザーがルームを選んで開始する個別の会話履歴 |
| プロジェクトユーザー | プロジェクトとルームを作成・編集できる制作者 |
| ユーザー | 公開されたルームで会話できる一般利用者 |
| スーパーユーザー | 全データを管理できる運営者 |

## 3. 基本構造
```text
Project
  └─ LiveChatRoom
       ├─ room_name
       ├─ conversation_objective_md
       ├─ character_id
       ├─ status
       └─ created_by_user_id

User
  └─ ChatSession
       ├─ room_id
       ├─ owner_user_id
       ├─ player_name
       ├─ room_snapshot_json
       ├─ chat_message
       ├─ session_state
       └─ session_image
```

ルームは会話の設計テンプレートであり、セッションはユーザーごとのプレイデータである。

## 4. 役割別仕様
### 4.1 プロジェクトユーザー
プロジェクトユーザーは、自分が管理するプロジェクト内でルームを作成・編集できる。

設定できる項目:
- ルーム名
- 会話の目的
- 話すキャラクター
- ルームの公開状態
- 必要であれば初期画像、初期状態、説明文

プロジェクトユーザーはルーム作成時にプレイヤー名を入力しない。プレイヤー名は実際に会話を開始するユーザー側の設定である。

プロジェクトユーザー自身がテストしたい場合は、ルーム詳細から「テスト会話を開始」を押し、自分用セッションを作成する。その時だけプレイヤー名を入力する。

### 4.2 ユーザー
ユーザーは、利用可能なプロジェクト内のルームを選んで会話を開始する。

ユーザーが入力できる項目:
- プレイヤー名

ユーザーが変更できない項目:
- 会話の目的
- 話すキャラクター
- ルーム名
- ルームの公開状態

ユーザーが作成したセッション、会話ログ、生成画像、評価状態、贈り物情報は、そのユーザー本人だけが閲覧できる。

### 4.3 スーパーユーザー
スーパーユーザーは全プロジェクト、全ルーム、全セッションを管理できる。ただし通常UIでは、一般ユーザーの会話本文を不用意に表示しない。調査・通報・サポート目的の閲覧は別導線にする。

## 5. ルームとセッションの関係
### 5.1 同じルームで複数セッションを作成できる
同じユーザーは、同じルームに対して複数のセッションを作成できる。

理由:
- 最初からやり直したい
- 別のプレイヤー名で試したい
- 違う会話ルートを見たい
- 途中で失敗した会話とは別に進めたい

制約:
- `room_id + owner_user_id` はユニークにしない
- 同一ルーム内で同一ユーザーが複数セッションを持てる
- セッション名は自動生成を基本とする

セッション名の初期値例:
- `ノアとの会話 2026-04-26 14:30`
- `ラプとの会話 #2`

ユーザー画面では、ルーム選択後に以下を選べるようにする。
- 既存セッションを続きから開く
- 新しく会話を始める

### 5.2 ルーム編集後の既存セッション
ルーム設定を変更しても、既存セッションの会話体験が途中で大きく変わらないようにする。

方針:
- セッション作成時に、ルーム設定のスナップショットを `chat_session.room_snapshot_json` に保存する
- 会話生成では原則としてスナップショットを使う
- ルーム変更は新規セッションに反映する
- 既存セッションへ反映したい場合は、将来的に「このセッションに最新ルーム設定を適用」機能を追加する

スナップショットに保存する情報:
- ルーム名
- 会話の目的
- キャラクターID
- キャラクター名
- ルーム設定バージョン

## 6. 画面仕様
### 6.1 プロジェクトユーザー向け ルーム一覧画面
目的:
プロジェクト内のライブチャットルームを管理する。

表示項目:
- ルーム名
- 話すキャラクター
- 会話の目的の要約
- 公開状態
- 更新日時
- ルームから作成されたセッション数

操作:
- 新規ルーム作成
- ルーム編集
- ルーム削除または非公開化
- テスト会話を開始

### 6.2 プロジェクトユーザー向け ルーム編集画面
目的:
会話テンプレートを作成・編集する。

入力項目:
- ルーム名: 必須
- 話すキャラクター: 必須
- 会話の目的: 必須、Markdown入力
- 公開状態: 下書き / 公開
- 説明文: 任意

入力しない項目:
- プレイヤー名

### 6.3 ユーザー向け ルーム選択画面
目的:
ユーザーが会話したいルームを選ぶ。

表示項目:
- ルーム名
- キャラクター名
- キャラクターサムネイル
- ルーム説明
- 自分の既存セッション数

操作:
- 既存セッションを開く
- 新しく会話を始める

### 6.4 ユーザー向け セッション開始画面
目的:
選択したルームから自分用セッションを作成する。

入力項目:
- プレイヤー名: 必須

自動設定:
- セッション名
- ルームID
- 話すキャラクター
- 会話の目的
- 初期状態

### 6.5 ライブチャット画面
目的:
ユーザーがキャラクターと会話する。

表示項目:
- メイン画像
- ノベル風テキストボックス
- 会話ログ
- 画像生成ボタン
- テキストボックス表示/非表示ボタン
- 画像候補
- 基準画像選択

編集できる項目:
- プレイヤー名は原則セッション開始時のみ
- 会話中にプレイヤー名変更が必要な場合は、別途セッション設定から変更可能にしてもよい

編集できない項目:
- 会話の目的
- 話すキャラクター

## 7. DB設計差分
### 7.1 追加テーブル: `live_chat_room`
| カラム | 型 | 必須 | 説明 |
|---|---|---:|---|
| `id` | integer | 必須 | 主キー |
| `project_id` | integer | 必須 | 所属プロジェクト |
| `created_by_user_id` | integer | 必須 | 作成したプロジェクトユーザー |
| `title` | string | 必須 | ルーム名 |
| `description` | text | 任意 | ユーザー向け説明 |
| `conversation_objective` | text | 必須 | Markdown形式の会話目的 |
| `character_id` | integer | 必須 | 話すキャラクター |
| `status` | string | 必須 | `draft`, `published`, `archived` |
| `sort_order` | integer | 必須 | 表示順 |
| `created_at` | datetime | 必須 | 作成日時 |
| `updated_at` | datetime | 必須 | 更新日時 |
| `deleted_at` | datetime | 任意 | 論理削除日時 |

インデックス:
- `INDEX(project_id)`
- `INDEX(character_id)`
- `INDEX(status)`
- `INDEX(project_id, status)`

### 7.2 変更テーブル: `chat_session`
追加カラム:
| カラム | 型 | 必須 | 説明 |
|---|---|---:|---|
| `room_id` | integer | 任意 | 元になったルーム。既存データ移行を考慮して一旦nullable |
| `owner_user_id` | integer | 必須 | セッション所有者 |
| `room_snapshot_json` | text/json | 任意 | セッション作成時点のルーム設定 |
| `player_name` | string | 必須 | ユーザーが入力するプレイヤー名 |

既存カラムの扱い:
- `title`: セッション名として残す。基本は自動生成
- `settings_json.conversation_objective`: 新規作成では使わない。ルームまたは `room_snapshot_json` へ移行する
- `settings_json.selected_character_id`: 新規作成では使わない。ルームまたは `room_snapshot_json` へ移行する

インデックス:
- `INDEX(room_id)`
- `INDEX(owner_user_id)`
- `INDEX(room_id, owner_user_id)`
- `INDEX(project_id, owner_user_id)`

ユニーク制約:
- `room_id + owner_user_id` にはユニーク制約を付けない

### 7.3 既存セッションの移行方針
既存の `chat_session` は、移行時にルームを自動作成する。

移行案:
1. 既存セッションごと、または同一プロジェクト・同一キャラクター・同一目的ごとに `live_chat_room` を作成
2. 既存 `settings_json.conversation_objective` を `live_chat_room.conversation_objective` にコピー
3. 既存 `settings_json.selected_character_id` を `live_chat_room.character_id` にコピー
4. `chat_session.room_id` を設定
5. `chat_session.room_snapshot_json` に当時のルーム設定を保存

MVPでは、既存セッションごとに1ルームを作る単純移行でよい。

## 8. API設計
### 8.1 ルームAPI
プロジェクトユーザー向け。

| メソッド | パス | 用途 |
|---|---|---|
| `GET` | `/api/v1/projects/{project_id}/chat/rooms` | ルーム一覧 |
| `POST` | `/api/v1/projects/{project_id}/chat/rooms` | ルーム作成 |
| `GET` | `/api/v1/chat/rooms/{room_id}` | ルーム詳細 |
| `PATCH` | `/api/v1/chat/rooms/{room_id}` | ルーム更新 |
| `DELETE` | `/api/v1/chat/rooms/{room_id}` | ルーム削除またはアーカイブ |

### 8.2 ユーザー向けルームAPI
一般ユーザー向け。

| メソッド | パス | 用途 |
|---|---|---|
| `GET` | `/api/v1/projects/{project_id}/chat/available-rooms` | 利用可能ルーム一覧 |
| `GET` | `/api/v1/chat/rooms/{room_id}/my-sessions` | 自分の既存セッション一覧 |
| `POST` | `/api/v1/chat/rooms/{room_id}/sessions` | ルームから新規セッション作成 |

`POST /api/v1/chat/rooms/{room_id}/sessions` の入力:
```json
{
  "player_name": "ぱぱぱ"
}
```

### 8.3 セッションAPI
既存APIは継続利用する。

| メソッド | パス | 用途 |
|---|---|---|
| `GET` | `/api/v1/chat/sessions/{session_id}` | セッション詳細 |
| `POST` | `/api/v1/chat/sessions/{session_id}/messages` | メッセージ送信 |
| `POST` | `/api/v1/chat/sessions/{session_id}/images/generate` | 画像生成 |
| `POST` | `/api/v1/chat/sessions/{session_id}/images/upload` | 画像アップロード |

変更点:
- 一般ユーザーはセッションの会話目的やキャラクターを更新できない
- プロジェクトユーザーも、他ユーザーのセッション本文は原則閲覧しない
- セッション作成はルーム経由を基本にする

## 9. 権限仕様
### 9.1 ルーム
| 操作 | スーパーユーザー | プロジェクトユーザー | ユーザー |
|---|---:|---:|---:|
| ルーム作成 | 可 | 自分のプロジェクトのみ可 | 不可 |
| ルーム編集 | 可 | 自分のプロジェクトのみ可 | 不可 |
| ルーム削除 | 可 | 自分のプロジェクトのみ可 | 不可 |
| 公開ルーム閲覧 | 可 | 可 | 利用可能プロジェクトのみ可 |

### 9.2 セッション
| 操作 | スーパーユーザー | プロジェクトユーザー | ユーザー |
|---|---:|---:|---:|
| 自分のセッション作成 | 可 | 可 | 可 |
| 自分のセッション閲覧 | 可 | 可 | 可 |
| 他ユーザーのセッション本文閲覧 | 原則可だが管理導線のみ | 原則不可 | 不可 |
| 他ユーザーのセッションメタ閲覧 | 可 | 自分のプロジェクト内は集計情報のみ可 | 不可 |

## 10. 会話生成への反映
会話生成時は、以下の優先順位で設定を取得する。

1. `chat_session.room_snapshot_json`
2. `live_chat_room`
3. 旧 `chat_session.settings_json`

新規セッションでは必ず `room_snapshot_json` を作成する。

AIプロンプトに渡す情報:
- プレイヤー名: `chat_session.player_name`
- 会話の目的: `room_snapshot_json.conversation_objective`
- 話すキャラクター: `room_snapshot_json.character_id`
- キャラクター設定
- 世界観設定
- 直近の会話ログ
- セッション状態

1. `live_chat_room` モデル、Repository、Serviceを追加
2. `chat_session` に `room_id`, `room_snapshot_json` を追加
3. ルームCRUD APIを追加
4. ルーム一覧・編集画面を追加
5. ユーザー向けルーム選択画面を追加
6. ルームからセッションを作成するAPIを追加
7. 既存セッション作成処理をルーム経由へ切り替え
8. ライブチャット画面から会話目的・話す相手の編集UIを削除または権限で非表示
9. 会話生成プロンプトを `room_snapshot_json` 参照へ変更
10. 既存データ移行マイグレーションを作成
