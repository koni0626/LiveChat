# ワールドマップ管理機能 仕様書

## 1. 目的

ワールド内に存在する場所・施設・区画・エリアを管理し、チャット、ストーリー、Feed、画像生成で再利用できるようにする。

本機能では、厳密な座標付き地図ではなく、ユーザーが「このワールドはこういう場所なんだ」と直感的に理解できる全体イメージを扱う。

対象は都市に限定しない。予備校、村、学園、会社、宇宙船、島、ダンジョンなど、あらゆるワールドに対応できる汎用機能とする。

## 2. 用語

### ワールドマップ

ワールド全体の雰囲気、主要な場所、空間構成を表す画像。

厳密な地図ではなく、俯瞰図、案内図、施設配置イメージ、鳥瞰イラスト、概念図に近い。

例:

- ラプラスシティの火星未来都市俯瞰図
- 予備校の校舎案内図
- 山間の村の全体イメージ
- ダンジョンの階層図
- 宇宙船の区画イメージ

### ロケーション

ワールド内の場所情報。

都市であれば施設、村であれば場所、学園であれば校内施設、ダンジョンであればエリアとして扱える。

画面表示名は当面「ロケーション」とする。

## 3. 管理対象

### ワールドマップ

管理する情報:

- ワールドID
- タイトル
- 説明
- 現在のワールドマップ画像Asset
- 生成/アップロード履歴
- 画像生成時のプロンプト
- 画像品質
- 画像サイズ
- ステータス

### ロケーション

管理する情報:

- ワールドID / project_id
- ロケーション名
- 種別
- 説明
- 所有者キャラクターID、任意
- 画像Asset、任意
- 根拠メモ
- 由来
- 表示順
- ステータス

種別の例:

- 統治
- 観測
- 研究
- 商業
- 娯楽
- 交通
- 生活
- 自然
- 宗教
- 教育
- 医療
- 金融
- ダンジョン
- 危険区域
- その他

由来の例:

- 手動登録
- Feedから抽出
- チャットから抽出
- ストーリーから抽出
- AI提案

## 5. 画面仕様

### ワールドホーム

ワールドホームに「ワールドマップ」ボタンを追加する。

ユーザーはワールドマップを閲覧できる。

プロジェクトユーザーと管理者は以下を操作できる。

- ワールドマップ管理を開く
- ロケーション一覧を開く

一般ユーザーは閲覧のみとし、登録、編集、削除、画像アップロード、画像生成はできない。

### ワールドマップ管理画面

表示内容:

- 現在のワールドマップ画像
- ワールドマップ説明
- 登録済みロケーション数
- 最終更新日時
- 画像アップロード
- ワールドマップ生成/更新
- 過去画像履歴

操作:

- 現在画像を拡大表示

プロジェクトユーザー/管理者のみ可能な操作:

- ワールドマップ画像をアップロード
- ワールドマップを生成
- ワールドマップを更新
- 過去画像から復元

### ロケーション一覧画面

表示内容:

- ロケーションカード一覧
- ロケーション画像
- ロケーション名
- 種別
- 所有者
- 説明の短縮表示
- ステータス

操作:

- 詳細表示

プロジェクトユーザー/管理者のみ可能な操作:

- 新規追加
- 編集
- 削除
- 表示順変更
- 画像アップロード
- 画像生成

### ロケーション編集画面

入力項目:

- ロケーション名
- 種別
- 説明
- 所有者キャラクター
- 根拠メモ
- ステータス
- 表示順
- ロケーション画像

説明と根拠メモはMarkdown入力を許可する。

## 6. DB設計案

### world_location

```text
world_location
- id
- project_id
- name
- location_type
- description
- owner_character_id
- image_asset_id
- source_type
- source_note
- status
- sort_order
- created_at
- updated_at
- deleted_at
```

補足:

- `project_id` はワールドに紐づく。
- `owner_character_id` はnullable。
- `image_asset_id` はnullable。
- `source_note` にはFeedやチャットから拾った根拠文を保存できる。
- `status` は `draft`, `published`, `archived` を想定。

### world_map_image

```text
world_map_image
- id
- project_id
- asset_id
- title
- description
- prompt_text
- source_type
- quality
- size
- is_active
- created_by_user_id
- created_at
- updated_at
- deleted_at
```

補足:

- アップロード画像も生成画像も同じテーブルで管理する。
- `is_active = 1` の画像を現在のワールドマップとして扱う。
- 画像履歴を残すため、古い画像は削除せず `is_active = 0` にする。

## 7. API設計案

### ワールドマップ

```text
GET    /api/v1/projects/<project_id>/world-map
POST   /api/v1/projects/<project_id>/world-map/upload
POST   /api/v1/projects/<project_id>/world-map/generate
POST   /api/v1/projects/<project_id>/world-map/images/<image_id>/select
DELETE /api/v1/projects/<project_id>/world-map/images/<image_id>
```

### ロケーション

```text
GET    /api/v1/projects/<project_id>/locations
POST   /api/v1/projects/<project_id>/locations
GET    /api/v1/locations/<location_id>
PATCH  /api/v1/locations/<location_id>
DELETE /api/v1/locations/<location_id>
POST   /api/v1/locations/<location_id>/image/upload
POST   /api/v1/locations/<location_id>/image/generate
```

権限:

- 閲覧はワールド閲覧権限。
- ワールドマップ自体は一般ユーザーも閲覧できる。
- ロケーション一覧/詳細も一般ユーザーが閲覧できる。
- 作成、更新、削除、画像アップロード、画像生成、現在マップ切り替え、過去画像復元はワールド管理権限。
- ワールド管理権限は、管理者または対象ワールドのプロジェクトユーザーに限定する。

## 8. 画像生成仕様

### ワールドマップ生成

入力:

- ワールド概要
- 世界観設定
- 登録済みロケーション一覧
- 現在のワールドマップ画像、任意
- ユーザー追加指示、任意

生成方針:

- 厳密な地図ではなく、ワールド全体の雰囲気が分かる全体イメージを生成する。
- 主要ロケーションが自然に存在しているように見せる。
- 文字ラベルの正確性には依存しない。
- ワールドのジャンルに応じて表現を変える。

例:

```text
このワールドの全体像を、俯瞰図または案内図風の一枚絵として生成する。
正確な座標や地図記号ではなく、「そういう世界なんだ」と分かる雰囲気を重視する。
登録済みロケーションを自然に含める。
```

### ロケーション画像生成

入力:

- ロケーション名
- ロケーション説明
- ワールド概要
- 所有者キャラクター情報、任意
- ワールドマップ画像、任意

生成方針:

- その場所単体の雰囲気が分かる画像を生成する。
- 所有者がいる場合は、そのキャラクターらしさを施設の内装や雰囲気に反映する。
- キャラクター本人を必ず描く必要はない。

## 9. チャット/ストーリーへの反映

チャットやストーリー生成時、同じワールドの公開ロケーションをプロンプトに含める。

含める内容:

- ロケーション名
- 種別
- 説明
- 所有者

利用例:

- 会話中の場所移動候補
- ストーリーの舞台候補
- Feed投稿の舞台
- 画像生成時の背景候補
- キャラクターの馴染みの場所

注意:

- 全ロケーションを毎回長文で渡すと重くなるため、MVPでは最大件数を制限する。
- 将来的には現在の会話内容に関連するロケーションだけを検索して渡す。

## 10. 将来機能: 自動抽出

将来的に、Feed投稿、チャット、ストーリーからロケーション候補を自動抽出する。

処理イメージ:

1. Feed/チャット/ストーリー本文を対象にする。
2. AIまたはルールベースで場所名候補を抽出する。
3. 既存ロケーションと重複判定する。
4. 候補一覧に表示する。
5. 管理者が採用/編集/破棄する。

候補として拾う例:

- 菩提樹はラプラスシティの避暑地
- ウォズ開発局
- ドル・バンク本店
- 観測塔
- 忘却回廊

自動登録は誤爆があり得るため、当面は「候補抽出」までにする。

## 11. MVP範囲

最初に実装する範囲:

- `world_location` テーブル
- `world_map_image` テーブル
- ロケーション一覧
- ロケーション作成/編集/削除
- ロケーション画像アップロード
- ワールドマップ画像アップロード
- 現在のワールドマップ表示
- ワールドホームからの導線

次に実装する範囲:

- ロケーション画像生成
- ワールドマップ生成/更新
- チャット/ストーリーへのロケーション反映

後回し:

- Feed/チャット/ストーリーからの自動抽出
- ロケーション候補レビュー画面
- ワールドマップ上のピン表示
- 座標管理

## 12. 実装メモ

- 既存の `AssetService` を画像保存に利用する。
- 画像Assetの `asset_type` は以下を想定する。
  - `world_map_image`
  - `world_location_image`
- ワールドマップ画像は履歴を残す。
- 現在画像は `world_map_image.is_active` で管理する。
- UIは既存のワールドホーム、ストーリー一覧、スタジオのカードデザインに寄せる。
- 一般ユーザーには閲覧のみ許可する。
- 登録、編集、削除、画像アップロード、画像生成、現在マップ切り替えはプロジェクトユーザー/管理者のみ許可する。
