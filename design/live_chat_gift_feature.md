# LiveChat 贈り物機能 仕様書

## 1. 目的
- プレイヤーが画像をアップロードして、キャラクターへ「贈り物」を渡せるようにする。
- AI が画像内容を認識し、キャラクターの好みや地雷と照合して反応・評価へ反映する。
- 贈り物イベントを会話ログと記憶に残し、以後の会話で参照できるようにする。
- 将来的には、キャラクターからプレイヤーへ物を渡す流れにも拡張できる構造にする。

## 2. 想定ユースケース
### 2-1. プレイヤー -> キャラクター
1. プレイヤーが会話画面から画像を選んでアップロードする。
2. その画像は会話ログ内にサムネイル付きで表示される。
3. AI が画像を解析し、「ぬいぐるみ」「服」「アクセサリ」などのラベルを推定する。
4. キャラクター設定の `likes / dislikes / hobbies / taboos / gift_preferences` と照合する。
5. キャラクターが反応セリフを返す。
6. 評価が上下する。
7. 贈り物イベントが記憶へ保存される。

### 2-2. キャラクター -> プレイヤー
1. 会話やイベントの流れで、キャラクターがプレイヤーへ物を渡す場合がある。
2. 渡された物はサムネイル付きで会話ログに残る。
3. プレイヤーは感想を返せる。
4. AI はその感想を「プレイヤーの好み」「その贈り物への印象」として記憶する。
5. キャラクターは後続会話でその記憶を参照する。

## 3. MVP 範囲
- プレイヤーが画像をアップロードして贈れる
- 会話ログにサムネイル表示
- AI による画像認識
- キャラクターの好みとの照合
- 評価の増減
- 贈り物イベントの保存
- 贈り物イベントを会話記憶に反映

## 4. 現在の v1 実装範囲
- `POST /api/v1/chat/sessions/{session_id}/gifts/upload` で画像アップロード可能
- AI が画像内容を認識して `recognized_label`, `recognized_tags` を推定
- キャラクター記憶と照合して `evaluation_delta` を計算
- 会話ログへサムネイル付きメッセージを追加
- キャラクターが贈り物に対する反応セリフを返す
- `session_gift_event` と `session_state.state_json.gift_memory` に保存
- `character.memory_profile_json.memorable_events` にも反映

## 5. 現在の v1 で未実装の範囲
- キャラクターがプレイヤーへ贈り物を渡す
- プレイヤーの感想を構造化して記憶する
- 贈り物を使った専用画像を自動生成する
- 「画像を出すべきかどうか」を AI が演出判断する

## 6. データモデル
### 6-1. `session_gift_event`
- `id`
- `session_id`
- `actor_type`
- `character_id`
- `asset_id`
- `gift_direction`
- `recognized_label`
- `recognized_tags_json`
- `reaction_summary`
- `evaluation_delta`
- `created_at`

### 6-2. `character.memory_profile_json`
```json
{
  "likes": ["ぬいぐるみ", "ガラス細工"],
  "dislikes": ["汚れた物"],
  "hobbies": ["服選び"],
  "taboos": ["怖い人形"],
  "gift_preferences": {
    "positive": ["ぬいぐるみ", "服", "アクセサリ"],
    "negative": ["壊れた物", "不気味な物"]
  }
}
```

### 6-3. `session_state.state_json.gift_memory`
```json
{
  "gift_memory": {
    "history": [
      {
        "character_name": "ノア",
        "recognized_label": "ぬいぐるみ",
        "recognized_tags": ["ぬいぐるみ", "白いクマ", "プレゼント"],
        "evaluation_delta": 16,
        "reaction_summary": "ノアはとても嬉しそうにしている。"
      }
    ],
    "last_gift": {
      "character_name": "ノア",
      "recognized_label": "ぬいぐるみ",
      "evaluation_delta": 16
    }
  }
}
```

## 7. AI 画像認識仕様
- 入力: プレイヤーがアップロードした画像
- 出力: JSON
- 想定キー:
  - `label`
  - `short_description`
  - `tags`
  - `likely_categories`
  - `confidence`

### 7-1. ラベル例
- `ぬいぐるみ`
- `ワンピース`
- `ネックレス`
- `花束`
- `お菓子`
- `本`

## 8. 評価ルール
### 8-1. プレイヤー -> キャラクター
- `gift_preferences.positive` 一致: 大きく加点
- `likes` 一致: 加点
- `hobbies` と関連: 中加点
- `gift_preferences.negative` 一致: 減点
- `dislikes` 一致: 減点
- `taboos` に関連: 大きく減点

### 8-2. キャラクター -> プレイヤー
- プレイヤーの感想が好意的なら加点
- 微妙・拒否寄りなら減点
- 感想内容はプレイヤー記憶へ保存する

## 9. 演出ルール
### 9-1. 基本方針
- 贈り物を渡したからといって、毎回かならず専用画像を出す必要はない。
- 会話の流れと演出効果を見て、「今この瞬間に画像を切り替えるべきか」を判断する。
- 画像が不要な場合は、会話ログと反応セリフだけを更新する。

### 9-2. 演出判断 AI の役割
贈り物イベント後に、演出判断 AI が以下を返す。

```json
{
  "show_gift_visual": true,
  "visual_priority": "high",
  "reason": "服を渡され、キャラクターが試着する場面は視覚的価値が高い",
  "visual_direction": "プレイヤーから贈られた服を気に入って、その場で着て見せている"
}
```

### 9-3. `show_gift_visual`
- `true`: 贈り物専用画像を生成する
- `false`: 画像は切り替えず、会話と評価だけ反映する

### 9-4. 画像化しやすい贈り物
- ぬいぐるみ
  - 抱きしめる
  - 頬に寄せる
  - ベッドや椅子に一緒に座らせる
- 服
  - 試着する
  - 着替え後の姿を見せる
- アクセサリ
  - 身につける
  - 鏡の前で見せる
- 花束
  - 胸元で抱える
  - テーブルや窓辺に飾る

### 9-5. 画像化しなくてもよい例
- 本
  - 今すぐ開く必要がない
- お菓子
  - 会話だけで喜びを返しても成立する
- 小物
  - リアクションだけで十分な場合がある

### 9-6. 会話との整合
- 専用画像を生成した場合、その後の会話は新しい見た目や状況を理解した状態で続ける。
- 例:
  - ぬいぐるみを抱いている画像を出したら、その次の会話では「抱きしめている」「気に入っている」を反映する。
  - 服を着た画像を出したら、その次の会話では「試着した」「似合うか気にしている」を反映する。

## 10. 期待する将来挙動
### 10-1. ぬいぐるみを贈った場合
- AI が画像を `ぬいぐるみ` と認識
- ノアがぬいぐるみ好きなら加点
- 必要なら `show_gift_visual = true`
- 新しい画像では、ノアがそのぬいぐるみを抱きしめている
- 以後の会話でも「そのぬいぐるみ」を覚えている

### 10-2. 服を贈った場合
- AI が画像を `服` と認識
- キャラクターが服に興味を持つ設定なら加点
- 必要なら `show_gift_visual = true`
- 新しい画像では、その服を着た姿になる
- 以後の会話で「似合ってる？」「気に入った」などが自然に出る
