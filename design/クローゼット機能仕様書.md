# クローゼット機能仕様書

## 目的

クローゼット機能は、キャラクターごとの衣装を永続的に管理し、チャット、ストーリー、おでかけ、メール、Feed、画像生成で共通参照できるようにする機能である。

現状はチャットセッション用の衣装ルームとストーリーセッション用の衣装ルームが別々に存在しており、衣装の正本が分散している。これをキャラクター単位のクローゼットに一本化し、同じキャラクターがどのサービスに登場しても一貫した衣装を使える状態にする。

## 基本方針

- 衣装データの正本はキャラクター単位の `character_outfit` に置く。
- チャットセッション専用衣装は廃止し、クローゼットへの作成・選択導線に変更する。
- ストーリーセッション専用衣装も廃止し、クローゼットへの作成・選択導線に変更する。
- 既存のチャット衣装・ストーリー衣装データは、初期実装では物理削除しない。
- 既存データは必要に応じて、後続タスクでクローゼットへ移行できるようにする。
- 画像生成時は、キャラクターの顔参照と衣装参照を分けて扱う。

## 対象範囲

### 初期実装に含める

- キャラクターごとの衣装一覧
- 衣装の新規登録
- 衣装画像のアップロード
- 衣装名、説明、タグ、用途、固定ルールの編集
- デフォルト衣装の設定
- 衣装の論理削除
- チャットからクローゼット衣装を選択する
- ストーリーからクローゼット衣装を選択する
- おでかけ開始時に衣装を選択する
- 画像生成時に選択衣装またはデフォルト衣装を参照画像として渡す

### 初期実装では後回し

- 既存チャット衣装・ストーリー衣装の自動移行
- AIによる衣装説明の自動解析
- 生成済み画像から衣装だけを抽出して保存する機能
- 季節やシーンに応じた自動衣装選択
- 複数キャラクターの衣装コーディネート提案

## 用語

| 用語 | 説明 |
| --- | --- |
| クローゼット | キャラクターに紐づく衣装管理領域 |
| 衣装 | 画像、説明、固定ルールを持つキャラクターの服装データ |
| デフォルト衣装 | 明示的な衣装選択がない場合に使用する衣装 |
| 選択衣装 | チャット、ストーリー、おでかけなどの文脈で今回使用する衣装 |
| 衣装参照画像 | 画像生成時に服装の基準として渡す画像 |
| 顔参照画像 | 画像生成時にキャラクターの顔・髪型の基準として渡す画像 |

## データ設計

### character_outfit

キャラクターごとの衣装を保存する。

| カラム | 型 | 説明 |
| --- | --- | --- |
| id | integer | 主キー |
| project_id | integer | プロジェクトID |
| character_id | integer | キャラクターID |
| name | varchar | 衣装名 |
| description | text | 衣装の説明 |
| asset_id | integer | 衣装参照画像のアセットID |
| thumbnail_asset_id | integer | サムネイル用アセットID |
| tags_json | text | タグ配列JSON |
| usage_scene | varchar | 用途。daily、date、story、battle、formal など |
| season | varchar | season指定。spring、summer、autumn、winter、all など |
| mood | varchar | 雰囲気。cute、elegant、casual、sexy、dark など |
| color_notes | text | 色や素材の補足 |
| fixed_parts | text | 必ず維持する要素 |
| allowed_changes | text | 変更してよい範囲 |
| ng_rules | text | 変更禁止、避ける表現 |
| prompt_notes | text | 画像生成プロンプトへ追加する補足 |
| is_default | boolean | デフォルト衣装か |
| status | varchar | active、archived |
| created_at | datetime | 作成日時 |
| updated_at | datetime | 更新日時 |
| deleted_at | datetime | 論理削除日時 |

### 制約

- `character_id` ごとに `is_default=true` の衣装は最大1件。
- `deleted_at` が入っている衣装は通常一覧に表示しない。
- `asset_id` は必須。テキストだけの衣装は初期実装では扱わない。
- プロジェクト外のキャラクターやアセットは参照できない。

## 画面仕様

### クローゼット一覧

URL例:

```text
/projects/<project_id>/closet
```

機能:

- キャラクター選択
- 選択キャラクターの衣装一覧表示
- デフォルト衣装の表示
- タグ、用途、季節での絞り込み
- 衣装追加ボタン
- 衣装編集
- 衣装削除
- デフォルトに設定

表示:

- 既存画面と同じ `panel-card`、`empty-panel`、ボタンデザインを使う。
- 衣装カードの高さは画像比率とメタ情報で安定させ、説明文で高さが大きく崩れないようにする。
- 長い説明は折りたたみ、詳細ボタンで展開する。

### 衣装追加・編集

入力項目:

- 衣装名
- 衣装画像
- 説明
- タグ
- 用途
- 季節
- 雰囲気
- 色・素材メモ
- 固定する要素
- 変更してよい範囲
- NGルール
- 画像生成用補足
- デフォルト衣装にするか

衣装画像アップロード:

- 画像ファイルを `Asset` として保存する。
- `asset_type` は `character_outfit_reference` とする。
- メタデータに `source=closet`、`character_id`、`outfit_id` を保持する。

## 既存機能の扱い

### チャット衣装ルーム

現状:

- チャットセッション単位で衣装候補を持つ。
- 選択した衣装は次のチャット画像生成の基準になる。

変更後:

- チャット内の衣装ルームは、キャラクターのクローゼット選択UIに置き換える。
- 「衣装を生成」は、チャット専用衣装ではなくキャラクター衣装を作成する導線にする。
- 「今回だけ使う」は必要ならセッション状態に `selected_outfit_id` を持たせる。
- 衣装そのものは必ず `character_outfit` に保存する。

### ストーリー衣装ルーム

現状:

- ストーリーセッション単位で衣装候補を持つ。
- 選択した衣装はストーリー画像生成の基準になる。

変更後:

- ストーリー内の衣装ルームは、クローゼット選択UIに置き換える。
- 「衣装を生成」は、ストーリー専用衣装ではなくキャラクター衣装を作成する導線にする。
- ストーリー画像生成時は、登場キャラクターごとに選択衣装またはデフォルト衣装を参照する。
- 旧ストーリー衣装APIは初期実装では非表示化し、後続で削除または移行する。

## サービス別連携

### チャット

- チャット画面で現在の衣装を表示する。
- 衣装選択はクローゼットから行う。
- チャット画像生成時は、選択衣装があればそれを使用する。
- 選択衣装がない場合はデフォルト衣装を使用する。
- デフォルト衣装もない場合は従来のキャラクター画像のみで生成する。

### ストーリー

- ストーリーセッションの登場キャラクターごとに衣装を選択できる。
- ストーリー画像生成時は、対象キャラクターの衣装参照画像を渡す。
- テキスト生成にも衣装名と説明を短く渡し、服装描写の一貫性を上げる。

### おでかけ

- おでかけ開始時に衣装を選択できる。
- 未選択の場合はデフォルト衣装を使用する。
- おでかけ画像生成時は、顔参照画像、衣装参照画像、ロケーション画像の順で参照を組み立てる。
- 完了メールに添付される画像も同じ衣装設定を反映する。

### メール

- メール本文生成時に、必要に応じて現在またはデフォルト衣装の情報を渡す。
- 添付画像生成がある場合は衣装参照画像を使う。

### Feed

- Feed画像生成時に衣装を選択できるようにする。
- 未選択時はデフォルト衣装を使う。

### スタジオ

- スタジオ画像編集時に「クローゼットへ保存」導線を追加できる。
- 初期実装では保存先としてクローゼットを選べる程度に留める。

## 画像生成プロンプト方針

画像生成では、顔と衣装を別の参照として明示する。

基本指示:

```text
Use the character face reference as the highest-priority identity reference.
Use the selected outfit reference as the highest-priority clothing reference.
Preserve the exact clothing design, silhouette, fabric impression, color palette, accessories, and distinctive details.
Do not replace the outfit with a different costume unless explicitly requested.
Minor pose and lighting adaptations are allowed.
```

日本語指定例:

```text
キャラクターの顔・髪型は顔参照画像を最優先に維持する。
衣装は選択された衣装参照画像を最優先に維持する。
服の形、色、素材感、装飾、特徴的なパーツを変えない。
明示的な指示がない限り、別の衣装に置き換えない。
ポーズ、光、背景に合わせた自然な微調整のみ許可する。
```

## API仕様案

### クローゼット

```text
GET    /api/v1/projects/<project_id>/outfits
GET    /api/v1/characters/<character_id>/outfits
POST   /api/v1/characters/<character_id>/outfits
GET    /api/v1/outfits/<outfit_id>
PATCH  /api/v1/outfits/<outfit_id>
DELETE /api/v1/outfits/<outfit_id>
POST   /api/v1/outfits/<outfit_id>/default
POST   /api/v1/outfits/<outfit_id>/image
```

### 選択状態

チャット、ストーリー、おでかけなどの文脈では、衣装そのものではなく `outfit_id` を保持する。

```text
POST /api/v1/chat/sessions/<session_id>/outfit
POST /api/v1/story-sessions/<session_id>/characters/<character_id>/outfit
```

おでかけは開始APIの payload に含める。

```json
{
  "character_id": 1,
  "location_id": 2,
  "mood": "romantic",
  "outfit_id": 10
}
```

## 移行方針

### 段階1

- `character_outfit` を追加する。
- クローゼット画面を追加する。
- 手動アップロードと編集を実装する。
- デフォルト衣装を画像生成に使えるようにする。

### 段階2

- チャット衣装ルームUIをクローゼット選択UIへ置き換える。
- チャット衣装生成APIを、クローゼット衣装作成APIへ寄せる。
- 旧チャット衣装APIはUIから呼ばれない状態にする。

### 段階3

- ストーリー衣装ルームUIをクローゼット選択UIへ置き換える。
- ストーリー衣装生成APIを、クローゼット衣装作成APIへ寄せる。
- 旧ストーリー衣装APIはUIから呼ばれない状態にする。

### 段階4

- 既存チャット衣装・ストーリー衣装をクローゼットへ取り込む移行ツールを用意する。
- 移行後、旧APIと旧UIを削除する。

## 実装メモ

- 旧機能をすぐ物理削除すると既存データ参照や画面遷移が壊れる可能性があるため、まずUIから外す。
- 画像生成の参照画像順序は、顔、衣装、場所の順を基本にする。
- 衣装名と説明はテキストAIにも渡すが、長い説明は要約してコンテキストを圧迫しない。
- クローゼット画面の `empty-panel` は既存画面と同じ見た目に揃える。
- 衣装カードは説明文を閉じた状態を標準にし、カード高さが説明文量で大きく変わらないようにする。

## 未決事項

- チャット内で「今回だけ使う」衣装を許可するか。
- ストーリーで複数キャラクターが登場する場合の衣装選択UIを、セッション開始時に置くか画像生成直前に置くか。
- 旧衣装データを自動移行するか、ユーザー操作で保存させるか。
- 生成した衣装画像を即デフォルトにする導線を入れるか。
- 衣装画像がないテキストのみ衣装を許可するか。
