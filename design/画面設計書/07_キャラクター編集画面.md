# キャラクター編集画面

## 1. 基本情報
- 画面名: キャラクター編集画面
- 画面ID: `screen-character-edit`
- 優先度: A
- 対象ユーザー: 制作者
- Bootstrap:
  - `Bootstrap 5.3`

## 2. 目的
- キャラクターの人格、口調、外見、画像固定ルールを編集する

## 3. 画面遷移
### 遷移元
- キャラクター一覧画面
- 作品編集トップ画面

### 遷移先
- キャラクター一覧画面

## 4. レイアウト
### 全体構成
- ヘッダー: キャラクター名
- サイドバー: 作品内メニュー
- メイン領域: 左 基本情報 / 中央 性格・口調 / 右 外見・画像固定ルール
- 補助領域: 画像プレビュー

### ブレークポイント方針
- `lg`: 3カラム
- `md`: 2カラム
- `sm`: 1カラムタブ化を検討

## 5. 表示項目
| 項目名 | 内容 | データ元 | 備考 |
|---|---|---|---|
| 基本情報 | 名前、役割、一人称、二人称など | `/api/v1/characters/<id>` |  |
| 外見情報 | appearance_summary、基準画像 | 同上 | 基準画像は reference image として利用 |
| 画像固定ルール | hair / face / ear / outfit 等 | `/api/v1/characters/<id>/image-rule` |  |

## 6. 入力項目
| 項目名 | 型 | 必須 | 初期値 | バリデーション | 備考 |
|---|---|---|---|---|---|
| name | text | 必須 | API値 | 空禁止 |  |
| role | text | 任意 | API値 | なし |  |
| age_impression | text | 任意 | API値 | なし |  |
| first_person | text | 任意 | API値 | なし |  |
| second_person | text | 任意 | API値 | なし |  |
| personality | textarea | 任意 | API値 | なし |  |
| speech_style | textarea | 任意 | API値 | なし |  |
| speech_sample | textarea | 任意 | API値 | なし |  |
| ng_rules | textarea | 任意 | API値 | なし |  |
| appearance_summary | textarea | 任意 | API値 | なし |  |
| base_asset_id | hidden/select | 任意 | API値 | なし | 画像選択と連動 |
| image_rule fields | text/select | 任意 | API値 | 一部値制限 |  |

## 7. 操作
| 操作 | トリガー | 成功時 | 失敗時 |
|---|---|---|---|
| 保存 | ボタン | 保存トースト | バリデーション表示 |
| 基準画像アップロード | ファイル選択 | `base_asset_id` 更新、プレビュー反映 | エラー表示 |
| 画像テスト生成 | ボタン | 画像生成確認画面 or モーダル | エラー表示 |
| AI補完 | ボタン | 対象入力欄へ反映 | エラー表示 |

## 8. 使用API
| 用途 | メソッド | パス | レスポンス利用箇所 |
|---|---|---|---|
| 詳細取得 | GET | `/api/v1/characters/<character_id>` | 初期表示 |
| 更新 | PATCH | `/api/v1/characters/<character_id>` | 保存 |
| 画像ルール取得 | GET | `/api/v1/characters/<character_id>/image-rule` | 初期表示 |
| 画像ルール更新 | PUT | `/api/v1/characters/<character_id>/image-rule` | 保存 |
| 基準画像アップロード | POST | `/api/v1/assets/upload` | `base_asset_id` 反映 |
| アセット一覧 | GET | `/api/v1/projects/<project_id>/assets` | 基準画像選択 |

## 9. 状態管理
- ローカル state: フォーム全体、dirty、saving、error
- URL クエリ: なし
- セッション: 認証セッション
- 保存タイミング: 明示保存

## 10. エラー・空状態
- ローディング: セクションごと skeleton
- 空状態: 画像未設定時プレースホルダ表示
- バリデーションエラー: 各入力欄に表示
- APIエラー: 上部アラート

## 11. デザインメモ
- Bootstrap コンポーネント: `accordion`, `card`, `form-control`, `form-check`, `nav-tabs`
- アイコン: 人物、画像、ルール
- トーン: 「キャラを作る楽しさ」を重視
